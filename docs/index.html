The problem

Every network is normal in its own way.

Signature-based intrusion detection can only catch attacks it has already seen. Anomaly detection learns normal behaviour and flags deviations — but "normal" differs sharply between organisations, and the raw traffic that would help a shared model is exactly the data no one can share.

◇
Unseen attacks

No signatures for novel threats

Learn the shape of benign traffic instead, and treat whatever fails to reconstruct as suspicious — no attack labels needed at training time.

≠
Non-IID data

Domain shift across silos

Services, traffic volume, user behaviour and attack mix all differ per organisation. A model update that helps one client can actively harm another.

⧉
Privacy

Raw NetFlow can't leave

Flow records expose internal infrastructure and communication patterns. Centralised pooling is off the table — so training has to come to the data.

The evidence

Four heterogeneous NetFlow datasets.

The detector is evaluated on four NetFlow-v3 intrusion-detection benchmarks that differ enormously in scale and class balance — from 0.3 % benign to 87 % benign. Each becomes one federated client.

The model

NetFlow in, reconstruction error out.

The core detector is a Graph Autoencoder. It rebuilds each network flow from the structure around it; a flow it can't rebuild well is one it has never really seen before — the anomaly score is simply how far the reconstruction missed.

01

Window → graph

Slice 1,000 consecutive flows into one graph. IP addresses become nodes; each flow an edge from source to destination.

02

Encode hosts

A graph-attention network plus a 3-level Graph U-Net turn each host into an embedding shaped by its neighbours.

03

Global context

All edge features in the window are mean-pooled through an MLP into one graph-level summary — context without exposing any single flow.

04

Decode flows

For each edge, the decoder reconstructs its features from the two host embeddings and the global context.

05

Score

Reconstruction MSE per flow is the anomaly score. High error → anomaly. The model is trained on benign traffic only.

Edge & node features

Seven raw fields per flow, 17-dim edge vectors.

Numeric features are z-scored on benign training statistics (optionally log-compressed); categorical fields are one-hot encoded with buckets for unseen values. Node vectors start from the IP's four octets, get randomly projected to 32 dims, then diffused three hops across the window's graph.

  • Flow duration, total packets, total bytes, min TTL — z-scored, 4 numeric
  • Protocol, src port, dst port — one-hot, top-k + buckets
  • Node = IP address — octet encoding → random projection → 3-hop diffusion, 32-dim
  • Training — AdamW, one-class on benign windows, early-stop on validation MSE
Federated learning

Train together. Keep your data.

Built on Flower, one client per dataset. Each round, clients pull the global model, train locally on private traffic, and return only parameters — the server averages them into the next global model. The open question: which parts to share, and which to keep local.

How the shared weights are learned — aggregation

FedAvg federated averaging

The server takes an equal-weighted average of the four clients' parameters. Simple and strong — but with very different data, local models can drift apart within a round.

FedProx proximal term

Adds a gentle penalty (μ = 0.01) that keeps each client from straying too far from the round's global weights. Tames drift — but here it didn't beat plain FedAvg.

What each client keeps private — personalisation

FedBN local batch-norm

BatchNorm parameters and running statistics stay on the client, absorbing each network's feature-distribution shift while the rest of the model is shared. The overall winner.

FedRep local head

The decoder and global-edge head stay private; only the encoder is aggregated. Clients alternate training head-then-body each round. Strong for some clients, uneven across others.

Results

Federation helps — when adaptation stays local.

Macro-averaged over all four datasets, the strongest configuration pairs FedAvg with FedBN: it posts the best ranking and F1 and the lowest false-positive rate. Sharing everything, or pooling data centrally, was consistently less reliable.

Cross-dataset generalisation

Does it transfer to a network it has never seen?

In a leave-one-dataset-out test, federated exposure to several environments improved held-out ranking for all four datasets versus single-source transfer — biggest gains on CICIDS (+31.6) and UNSW (+22.3). But high false-positive rates on BoT-IoT and CICIDS show the domain gap shrinks, it doesn't close: a new organisation still needs local calibration.

Privacy mechanisms

Two defences, two very different price tags.

Keeping raw data local isn't automatically private — model updates leak too. The project adds two protections, and measures what each one costs in detection quality.

≈ free

Secure aggregation · SecAgg+

Pairwise masks cancel in the sum, so an honest-but-curious server only ever sees the total of the four updates, never any single one. Up to quantisation it computes the exact same average — so detection performance is unchanged. Enable it by default.

steep trade-off

Central differential privacy

Server-side clipping (C = 0.13) plus Gaussian noise makes the released model provably insensitive to any one client. But with only four clients there's no subsampling amplification — useful utility is gone while the privacy budget ε is still in the hundreds.

Deployability

Fast enough to run on commodity CPUs.

Benchmarked on the 1.22M-flow UNSW holdout. The 323K-parameter model is so light that on a GPU the bottleneck is building the graphs, not running the network — and even CPU-only clears 90,000 flows per second inside 4 GB of RAM.

Run it

Two ways to train the detector.

One centralised script trains and evaluates every dataset in a single process; the federated workflow runs one client per dataset over Flower. Full flags and the smoke-test recipe are in the README.

centralised · train_and_inferv2.py
# set up the environment
conda env create -f environment.yml
conda activate gnn-netflow-ad

# one model per dataset (default)
python train_and_inferv2.py

# shared encoder + per-dataset head
python train_and_inferv2.py --personalised-frozen
federated · run_federated_gpu.sh
# best config: FedAvg + FedBN + SecAgg+
./run_federated_gpu.sh \
  --shared_weight=FedAvg \
  --personalisation=FedBN \
  --secagg=true --clip=8.0 \
  --rounds=150 --seeds="42 43 44"
